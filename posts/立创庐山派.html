立创庐山派（K230-CanMV）教学文档

前言

立创庐山派-K230-CanMV开发板是立创开发板团队精选打造的超高性价比AI视觉开发板，以嘉楠科技Kendryte®系列最新一代SoC芯片K230为主控，具备强劲的AI推理能力和便捷的开发体验，支持CanMV平台与MicroPython编程，无需复杂的底层配置，即可快速开展AI视觉、边缘计算相关项目开发，是新手入门AI开发、参加电子设计竞赛的优质选择。

本文将从基础认知到实操落地，逐步引导大家掌握立创庐山派的核心使用方法，内容兼顾理论与实操，重点突出步骤细节和避坑要点，全程配合实例演示，确保不同基础的开发者都能快速上手。

第一章 开发板简介

1.1 核心定位

立创庐山派是一款以AI视觉为核心的开发板，主打“高性价比、低门槛、强性能”，可作为AI与边缘计算平台，广泛应用于物联网、智能家居与消费电子、工业自动化、无人系统等领域，典型网络下的推理能力可达K210的13.7倍，能轻松应对图像识别、目标检测等常见AI开发需求。

1.2 核心硬件参数

主控芯片采用K230，作为全球首款支持RISC-V Vector 1.0标准的商用SOC，其核心参数如下，配套开发板硬件资源丰富，可满足多样化开发需求：

  • 主控芯片：K230（嘉楠科技），12nm制程工艺，主频高达1.6GHz，集成两颗RISC-V玄铁C908核心；
  • AI算力：配备第三代KPU处理单元，等效6TOPS算力，专为图像、视频、音频处理和AI加速设计；
  • 图像输入：支持三路MIPI CSI视频输入，最大分辨率可达4K，兼容GC2093、OV5647等摄像头（22P 0.5mm间距）；
  • 存储与内存：支持TF卡扩展（建议2G以上class10等级），板载1G内存；
  • 编程支持：支持CanMV平台，可直接使用MicroPython编程，兼容TensorFlow、PyTorch等常见AI计算框架；
  • 其他资源：板载麦克风、蜂鸣器、耳机接口，配备Type-C供电/通信接口，支持WiFi模块扩展，预留多路ADC接口及显示屏接口。

1.3 开发板外观与接口说明

开发板整体布局清晰，接口标注明确，核心接口如下（新手重点关注以下接口，避免误插损坏硬件）：

  • Type-C接口：核心供电与通信接口，用于连接电脑（烧录程序、调试）和供电；
  • TF卡插槽：用于插入烧录好固件的TF卡，为开发板提供系统运行环境；
  • 摄像头接口（CSI）：共3路，用于连接兼容的摄像头模块，开展AI视觉相关开发；
  • LED指示灯：板载RGB三色LED（红、绿、蓝），用于直观判断开发板运行状态、调试程序；
  • 扩展接口：预留多路GPIO、ADC接口，可连接传感器、舵机等外设，扩展项目功能。

提示：开发板到手后需先完成组装（参考官方组装视频），安装散热片、摄像头等配件时，注意接口对齐，避免用力过猛损坏针脚。

第二章 开发前准备

开发前需准备好相关硬件和软件，确保后续实操顺利进行，新手建议严格按照以下清单准备，避免因配件缺失或版本不兼容导致操作失败。

2.1 硬件准备

核心硬件清单（标准版开发板标配部分配件，其余需自行准备）：

序号 名称 规格要求 备注
1 立创庐山派主板 K230-CanMV版本 核心硬件，标准版标配
2 摄像头模块 GC2093或OV5647（22P 0.5mm间距） 标准版标配GC2093，用于AI视觉采集
3 TF卡 2G以上，class10等级及以上 建议4G或8G（无需大容量），推荐立创商城MK 8G TF卡
4 Type-C数据线 支持数据传输（非仅充电线） 用于连接电脑，传输程序、供电
5 TF卡读卡器 支持TF卡读写 用于将固件烧录到TF卡中
6 电脑 Windows系统（推荐Win10/11） 用于安装开发环境、烧录固件、编写程序

可选配件：WiFi模块、显示屏、杜邦线、传感器、开发板专用支架（需额外搭配铰链、螺丝等五金件）。

2.2 软件准备

需安装/下载以下4类软件/工具，全部选择最新版本，避免版本兼容问题：

  1. CanMV IDE K230：开发板专属编程与调试工具，用于编写MicroPython程序、调试代码、查看摄像头画面； 下载地址：嘉楠开发者社区-资料下载栏目（优先），选择对应Windows系统版本，写本文时最新版本为4.0.9。
  2. 固件文件：立创庐山派专用固件（带LCKFB字符），用于驱动开发板运行，需下载.img格式镜像文件； 下载渠道：推荐从嘉楠开发者社区获取（最新最稳定），或Github仓库（Releases · kendryte/canmv_k230）。
  3. 固件烧录工具Rufus：免安装软件，用于将固件烧录到TF卡中； 下载地址：https://rufus.ie/downloads/，或从离线资料包中获取（版本4.5及以上即可）。
  4. 解压工具：如WinRAR、7-Zip，用于解压下载的固件压缩包（固件下载后多为.gz后缀压缩包）。

警告：严禁下载非立创庐山派专用固件（不带LCKFB字符），严禁直接烧录.gz后缀压缩包，否则可能导致硬件损坏。

第三章 固件获取与烧录（核心步骤）

固件是开发板运行的基础，需先将对应固件烧录到TF卡中，再插入开发板，此步骤是新手最易出错的环节，需严格按照步骤操作。

3.1 固件获取与解压

  1. 打开嘉楠开发者社区资料下载页面（推荐渠道），依次点击「资料下载」→「K230」→「Images」→「CanMV」→「Micropython」；
  2. 选择带有「LCKFB」字符的最新版本固件，优先下载「Micropython Daily Build」版本（每日自动编译，修复已知BUG）；
  3. 下载完成后，找到.gz后缀的压缩包，右键使用解压工具解压，得到.img后缀的固件镜像文件（解压后仅保留.img文件，压缩包可删除）；
  4. 验证固件：确认解压后的文件后缀为.img，文件名包含「LCKFB」和「micropython」，避免解压错误。

3.2 固件烧录（使用Rufus工具）

  1. 将TF卡插入读卡器，再将读卡器连接到电脑，确保电脑能正常识别TF卡（可在“此电脑”中查看盘符）；
  2. 打开Rufus工具（免安装，双击即可运行），工具会自动识别插入的TF卡（确认设备列表中的盘符与TF卡一致，避免选错设备格式化其他磁盘）；
  3. 点击Rufus工具中「镜像文件（请选择）」后方的「选择」按钮，找到解压后的.img固件文件，选中并打开；
  4. 确认参数：无需手动修改任何参数，保持工具默认设置即可；
  5. 开始烧录：点击工具右下角的「开始」按钮，弹出警告提示（提示TF卡中所有数据将被清除），确认TF卡中无重要数据后，点击「确定」；
  6. 等待烧录完成：烧录过程中，工具状态栏会显示进度条，等待进度条走完，提示「准备就绪」，即表示固件烧录成功；
  7. 后续操作：关闭Rufus工具，在电脑上安全弹出读卡器，拔出TF卡（注意：不要直接拔掉读卡器，避免固件损坏）。

提示：烧录过程中不要断开读卡器与电脑的连接，不要操作TF卡，若烧录失败，可重新解压固件，更换TF卡或读卡器后重试。

第四章 开发环境搭建（CanMV IDE K230）

CanMV IDE K230是立创庐山派的专属开发环境，用于编写、调试MicroPython程序，搭建过程简单，按照以下步骤操作即可。

4.1 软件安装

  1. 找到下载好的CanMV IDE K230安装包（.exe格式），双击打开安装程序；
  2. 按照安装向导提示操作，点击「下一步」，选择安装路径（建议不要安装到C盘，如D:\CanMV IDE K230）；
  3. 继续点击「下一步」，勾选“创建桌面快捷方式”（方便后续打开），点击「安装」；
  4. 等待安装完成（约1-2分钟），安装完成后点击「完成」，关闭安装向导。

4.2 软件启动与验证

  1. 启动软件：双击桌面“CanMV IDE K230”快捷方式，或在电脑搜索栏中搜索“CanMV IDE K230”（注意区分K210版本，仅K230标识的软件兼容本开发板）；
  2. 验证安装：软件启动后，界面会显示默认的MicroPython模板代码，若能正常显示界面，无报错提示，即表示安装成功；
  3. 若无法找到软件：可进入安装路径下的「bin」文件夹，找到「canmvide.exe」文件，双击启动即可。

第五章 开发板启动与连接

完成固件烧录和开发环境搭建后，即可启动开发板，并将其与电脑、开发环境连接，准备编写调试程序。

5.1 开发板启动

  1. 确保开发板处于未供电状态（未连接Type-C数据线）；
  2. 将烧录好固件的TF卡，正确插入开发板的TF卡插槽（注意方向，插入到位，避免松动）；
  3. 用Type-C数据线，一端连接开发板的Type-C接口，另一端连接电脑的USB接口，为开发板供电并建立通信；
  4. 观察启动状态：若启动正常，开发板上的红色LED指示灯会自动点亮，等待3-10秒（第一次上电会稍慢）；
  5. 启动验证：电脑会自动识别开发板，设备管理器中会出现「USB串行设备（COMx）」，同时电脑中会出现CanMV设备（可当作U盘访问，包含固件自带的示例程序）。

注意：第一次上电时，开发板会自动将TF卡除固件外的剩余空间格式化为fat格式，并挂载到/data文件夹，此过程会自动重启一次，属于正常现象，后续上电不会再出现。

5.2 开发板与CanMV IDE连接

  1. 确保开发板已正常启动，电脑已识别开发板；
  2. 打开CanMV IDE K230软件，观察软件左下角的连接图标，若图标右上角出现USB标志，说明软件已检测到开发板；
  3. 点击左下角的USB连接图标，即可建立软件与开发板的连接；
  4. 连接成功验证：软件右下角的三角形运行按钮会变为绿色，同时串口终端会显示开发板的启动信息，表明连接成功，可以开始编写调试程序。

提示：若未检测到开发板，可检查Type-C数据线是否支持数据传输、TF卡是否插好、固件是否烧录正确，或重启电脑、重新插拔开发板后重试。

第六章 基础实操案例（新手必练）

本节通过2个基础案例，帮助新手熟悉开发板的使用方法和编程逻辑，案例难度从易到难，重点掌握核心API的使用，后续可基于此扩展复杂项目。

案例1：点亮板载LED灯（入门必练）

本案例实现板载红色LED灯闪烁，掌握Pin引脚的控制方法，验证开发板和开发环境是否正常工作。

1. 案例原理

立创庐山派板载RGB三色LED灯（红、绿、蓝），对应引脚分别为Pin62（红灯）、Pin20（绿灯）、Pin63（蓝灯），LED灯为共阳结构，引脚设置为低电平时点亮，高电平时关闭。通过控制引脚的高低电平切换，实现LED灯闪烁。

2. 编写代码

  1. 打开CanMV IDE K230，新建一个Python文件（点击菜单栏「File」→「New File」）；
  2. 将以下代码复制到编辑区，代码中包含详细注释，可根据注释理解每一行的作用：
from machine import Pin
import time

# 实例化Pin引脚，分别控制红、绿、蓝三个LED灯，设置为输出模式
LED_R = Pin(62, Pin.OUT, pull=Pin.PULL_NONE, drive=7)  # 红灯（核心控制引脚）
LED_G = Pin(20, Pin.OUT, pull=Pin.PULL_NONE, drive=7)  # 绿灯
LED_B = Pin(63, Pin.OUT, pull=Pin.PULL_NONE, drive=7)  # 蓝灯

# 初始化：关闭所有LED灯（共阳结构，高电平关闭）
LED_R.high()
LED_G.high()
LED_B.high()

# 主循环：实现红灯闪烁（点亮0.5秒，关闭0.5秒，循环执行）
while True:
    LED_R.low()          # 点亮红灯
    time.sleep(0.5)      # 延时0.5秒
    LED_R.high()         # 关闭红灯
    time.sleep(0.5)      # 延时0.5秒

3. 运行程序

  1. 保存文件：点击菜单栏「File」→「Save」，将文件保存到电脑（建议保存到容易找到的路径，文件名如“led_blink.py”）；
  2. 确认开发板与软件已连接（右下角运行按钮为绿色）；
  3. 点击软件右下角的绿色运行按钮，或按下快捷键F5，运行程序；
  4. 观察效果：开发板上的红色LED灯会开始闪烁，每0.5秒切换一次状态，说明程序运行成功。

扩展练习：修改代码中的引脚（如改为LED_G），实现绿灯闪烁；修改time.sleep()中的参数，改变闪烁频率。

案例2：摄像头画面采集与显示

本案例实现摄像头画面采集，并在CanMV IDE中显示，掌握摄像头的初始化和画面采集方法，为后续AI视觉开发打下基础。

1. 案例准备

  1. 将摄像头模块（如GC2093）正确连接到开发板的CSI接口（接口对齐，轻轻按压到位）；
  2. 确保开发板已正常启动，且与CanMV IDE连接成功；
  3. 确认固件支持摄像头（下载的带LCKFB字符的固件，默认支持摄像头）。

2. 编写代码

import sensor, image, time

# 初始化摄像头
sensor.reset()                          # 重置摄像头
sensor.set_pixformat(sensor.RGB565)     # 设置像素格式为RGB565（彩色）
sensor.set_framesize(sensor.QVGA)       # 设置画面尺寸为QVGA（320x240），适配开发板性能
sensor.skip_frames(time=2000)           # 跳过2000毫秒（2秒），等待摄像头稳定
sensor.set_auto_gain(False)             # 关闭自动增益（避免画面过亮/过暗）
sensor.set_auto_whitebal(False)         # 关闭自动白平衡（保持颜色稳定）

# 主循环：采集并显示摄像头画面
while True:
    img = sensor.snapshot()             # 采集一帧画面
    # 画面显示：CanMV IDE中会自动在右上角显示摄像头画面，无需额外代码
    time.sleep(10)                      # 延时10毫秒，避免画面卡顿

3. 运行程序与效果

  1. 保存文件（如“camera_display.py”），点击绿色运行按钮，运行程序；
  2. 观察效果：CanMV IDE右上角会出现摄像头采集的实时画面，若画面模糊，可调整摄像头焦距（旋转摄像头镜头）；
  3. 若未显示画面：检查摄像头连接是否正确、固件是否支持、画面尺寸设置是否合理，重启开发板后重试。

扩展练习：修改sensor.set_framesize()中的参数，改变画面尺寸；添加img.draw_string()函数，在画面上显示文字。

第七章 常见问题排查（新手必看）

新手在操作过程中，容易遇到固件烧录失败、开发板无法识别、程序运行报错等问题，以下汇总常见问题及解决方案，快速排查故障。

7.1 固件烧录相关问题

  • 问题1：Rufus工具无法识别TF卡？ 解决方案：检查读卡器与电脑连接是否正常，更换读卡器或TF卡；确认TF卡已格式化（格式为FAT32）；重启电脑后重试。
  • 问题2：烧录完成后，开发板无法启动（LED灯不亮）？ 解决方案：确认固件为带LCKFB字符的.img文件，未直接烧录.gz压缩包；检查TF卡插入是否到位；更换TF卡，重新烧录固件。
  • 问题3：烧录过程中提示“设备被占用”？ 解决方案：关闭电脑中占用TF卡的软件（如“此电脑”中的TF卡盘符窗口）；重新插拔读卡器，重试烧录。

7.2 开发板连接相关问题

  • 问题1：CanMV IDE无法检测到开发板（无USB标志）？ 解决方案：检查Type-C数据线是否支持数据传输（更换数据线重试）；确认开发板已正常启动（LED灯亮）；在设备管理器中检查是否有未知设备，安装对应驱动；重启软件或电脑。
  • 问题2：连接开发板后，运行按钮仍为灰色？ 解决方案：重新插拔Type-C数据线；确认固件烧录正确，开发板已正常启动；关闭软件，重新启动后连接。
  • 问题3：第一次上电启动缓慢，且自动重启？ 解决方案：属于正常现象，第一次上电会格式化TF卡剩余空间，等待重启完成后，后续上电会恢复正常速度。

7.3 程序运行相关问题

  • 问题1：运行LED灯程序，LED灯不闪烁？ 解决方案：检查代码中的引脚是否正确（红灯为Pin62）；确认LED灯为共阳结构，代码中高低电平设置正确；重新运行程序，或重启开发板。
  • 问题2：运行摄像头程序，无画面显示？ 解决方案：检查摄像头连接是否正确（CSI接口对齐）；确认固件支持摄像头；修改代码中的画面尺寸，降低分辨率；调整摄像头焦距，或更换摄像头。
  • 问题3：程序运行报错，提示“no module named 'machine'”？ 解决方案：确认烧录的固件为MicroPython固件（带micropython字符）；重新烧录固件，确保固件版本与软件版本兼容。
